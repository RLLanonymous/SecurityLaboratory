## Reporting a Vulnerability

To report a security vulnerability, please follow these steps:

1. **Report via Discord**: open a ticket and send a message with the info below.
   
2. **Details to include**:
   - **Vulnerability level (low, medium, high, critical)**
   - **Description of the vulnerability (including console logs)**
   - **Project version**
   - **SCP:SL version**
   - **Exiled version**
   - **Steps to reproduce, if applicable**
   - **Impact assessment (what can happen if exploited)**

4. **Response time**: We will acknowledge receipt of your report within **48 hours**. We aim to provide a fix or patch within **two weeks**, depending on the severity of the vulnerability.

5. **Process after reporting**:
   - If the vulnerability is valid, we will assign a CVE (Common Vulnerabilities and Exposures) number, release a patch, and notify users about the update.
   - If the report is not deemed a vulnerability, we will notify you with an explanation.

6. **Confidentiality**: Please avoid publicly disclosing vulnerabilities until a patch has been released. We will ensure to handle vulnerabilities with the highest priority and confidentiality.

7. **Thanks**: We appreciate and encourage responsible disclosure to help keep our users and project secure.

## Supported Versions

We strongly recommend using supported versions to ensure you receive timely security patches. If you're using an unsupported version, we encourage you to upgrade to the latest supported release as soon as possible.
